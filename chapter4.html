4.1 Introduction - What is Decision Making?

Programs don't always execute every statement.
Sometimes they must decide:

  • Is the number positive?
  • Is age greater than 18?
  • Is the user logged in?
  • Which menu option did the user choose?
  • Is password correct?
Decision making allows the program to choose different paths.

Example 
If it is raining
    Take umbrella
Else
    Go normally

Programming works exactly the same way.

Flow Diagram

Condition ?
↓
True
↓
Execute Block A
False
↓
Execute Block B
💡 Memory Tip
Decision Making means choosing one path from multiple possible paths based on a condition.
4.2 if Statement

The if statement executes a block of code only when the specified condition evaluates to true. If the condition is false, the statements inside the if block are skipped.

Syntax

if(condition)
{
    statements;
}

Flow

Condition ?
↓
True
↓
Execute Statements
False
↓
Skip Statements

Example

#include <stdio.h>

int main()
{
    int age = 20;

    if(age >= 18)
    {
        printf("Eligible to Vote");
    }

    return 0;
}
Eligible to Vote
Another Example 
int num = -5;

if(num > 0)
{
    printf("Positive Number");
}
No Output
💡 Remember
  • If condition is TRUE → Execute block.
  • If condition is FALSE → Skip block.
4.3 if-else Statement

The if-else statement is used when there are exactly two possible outcomes. One block executes when the condition is true, and the other executes when it is false.

Syntax

if(condition)
{
    statements;
}
else
{
    statements;
}

Example

#include <stdio.h>

int main()
{
    int age = 15;

    if(age >= 18)
    {
        printf("Adult");
    }
    else
    {
        printf("Minor");
    }

    return 0;
}
Minor
Example: 
int number = 10;

if(number % 2 == 0)
{
    printf("Even");
}
else
{
    printf("Odd");
}
Even
💡 Tip Only one block (if or else) will execute.
4.4 else-if Ladder

The else-if ladder is used when there are multiple conditions to check. The program evaluates each condition one by one. As soon as one condition is true, its corresponding block is executed and the remaining conditions are skipped.

Syntax

if(condition1)
{
    statements;
}
else if(condition2)
{
    statements;
}
else if(condition3)
{
    statements;
}
else
{
    statements;
}

Flow Diagram

Condition 1 ?
↓
True
↓
Execute Block 1
↓
False → Check Condition 2
↓
Continue Until Match

Example

#include <stdio.h>

int main()
{
    int marks = 82;

    if(marks >= 90)
    {
        printf("Grade A");
    }
    else if(marks >= 75)
    {
        printf("Grade B");
    }
    else if(marks >= 50)
    {
        printf("Grade C");
    }
    else
    {
        printf("Fail");
    }

    return 0;
}
Grade B
How it Works
  • Check if marks are 90 or above.
  • If not, check if marks are 75 or above.
  • If not, check if marks are 50 or above.
  • If none of the conditions are true, execute the else block.
💡 Memory Tip Only the first true condition is executed. All remaining conditions are skipped.
4.5 Nested if

A Nested if is an if statement placed inside another if statement. It is useful when one condition should be checked only after another condition becomes true.

Syntax

if(condition1)
{
    if(condition2)
    {
        statements;
    }
}

Flow Diagram

Age ≥ 18 ?
↓
Yes
↓
Citizen ?
↓
Yes
↓
Eligible to Vote
No
↓
Not Eligible

Example

#include <stdio.h>

int main()
{
    int age = 22;
    int citizen = 1;

    if(age >= 18)
    {
        if(citizen == 1)
        {
            printf("Eligible to Vote");
        }
    }

    return 0;
}
Eligible to Vote
Explanation

The second if statement is executed only if the first condition (age >= 18) is true.

💡 Memory Tip Nested if means checking another condition inside an existing if block.
4.6 switch Statement

The switch statement is used to execute one block of code from multiple choices. It compares the value of an expression with different case labels. When a match is found, the corresponding block is executed.

Note: The switch statement is generally used when comparing one variable against multiple fixed values.

Syntax

switch(expression)
{
    case value1:
        statements;
        break;

    case value2:
        statements;
        break;

    ...

    default:
        statements;
}

Flow Diagram

Evaluate Expression
↓
Compare with Cases
↓
Case Match
↓
Execute Statements
↓
break
No Match
↓
default

Example

#include <stdio.h>

int main()
{
    int day = 3;

    switch(day)
    {
        case 1:
            printf("Monday");
            break;

        case 2:
            printf("Tuesday");
            break;

        case 3:
            printf("Wednesday");
            break;

        default:
            printf("Invalid Day");
    }

    return 0;
}
Wednesday

Why is break Important?

The break statement terminates the current case. Without it, execution continues into the next case. This is known as fall-through.

Without break

switch(num)
{
    case 1:
        printf("One");

    case 2:
        printf("Two");

    case 3:
        printf("Three");
}
Output (num = 2)

Two
Three
Remember: Without break, all subsequent cases execute until a break or the end of the switch statement is reached.
💡 Memory Tip
  • switch is ideal for menus and fixed choices.
  • Always use break unless fall-through is intended.
  • default is optional but recommended.
4.7 Conditional (Ternary) Operator

The Conditional Operator, also called the Ternary Operator (?:), is a compact way of writing a simple if-else statement.

Syntax

condition ? expression1 : expression2;

If the condition is true, expression1 is executed. Otherwise, expression2 is executed.

Flow Diagram

Condition ?
↓
True
↓
Expression 1
False
↓
Expression 2

Example

#include <stdio.h>

int main()
{
    int a = 15;

    (a > 10) ? printf("Greater")
             : printf("Smaller");

    return 0;
}
Greater

Another Example

int x = 5;
int result;

result = (x > 0) ? 1 : 0;

printf("%d", result);
1
The ternary operator should be used only for short and simple conditions. For complex logic, prefer if-else for better readability.
💡 Memory Tip 
condition ? True : False
Think of it as a one-line if-else statement.
4.8 Difference between if and switch

Both if and switch are decision-making statements. However, they are used in different situations.

if Statementswitch Statement
Works with conditions.Works with fixed values.
Supports relational operators (>, <, ==, !=, etc.).Compares only equality with case values.
Suitable for complex conditions.Suitable for menu-driven programs.
Can evaluate multiple expressions.Evaluates only one expression.
May become lengthy with many conditions.Cleaner when there are many fixed choices.

Example using if

if(age >= 18)
{
    printf("Eligible to Vote");
}

Example using switch

switch(choice)
{
    case 1:
        printf("Add");
        break;

    case 2:
        printf("Delete");
        break;

    default:
        printf("Invalid Choice");
}
💡 Memory Tip
  • Use if for ranges and conditions.
  • Use switch for fixed menu options.
4.9 Common Programming Examples

1. Check Positive or Negative

if(num >= 0)
    printf("Positive");
else
    printf("Negative");

2. Check Even or Odd

if(num % 2 == 0)
    printf("Even");

3. Find Largest of Two Numbers

if(a > b)
    printf("%d", a);
else

4. Voting Eligibility

if(age >= 18)
    printf("Eligible");
else
    printf("Not Eligible");

5. Calculator using switch

switch(choice)
{
    case '+':
        printf("%d", a+b);
        break;

    case '-':
        printf("%d", a-b);
        break;

    case '*':
        printf("%d", a*b);
        break;

    case '/':
        printf("%d", a/b);
        break;

    default:
        printf("Invalid Operator");
}
These examples are commonly asked in lab exams and interviews.
4.10 Common Mistakes

1. Using = instead of ==

Wrong

if(age = 18)

Correct

if(age == 18)

2. Forgetting break in switch

Wrong

case 1:
    printf("One");

Correct

case 1:
    printf("One");
    break;

3. Missing Braces

Wrong

if(a > b)
    printf("A");
    printf("Done");

Correct

if(a > b)
{
    printf("A");
    printf("Done");
}
Always use braces even for a single statement. It improves readability and prevents bugs.
4.11 Best Practices
  • ✔ Always use braces { }.
  • ✔ Keep conditions simple and readable.
  • ✔ Use meaningful variable names.
  • ✔ Prefer switch for menu-driven programs.
  • ✔ Avoid deeply nested if statements.
  • ✔ Include a default case in every switch statement.
  • ✔ Indent your code properly.
